História da Web

Início

A história da computação - não da web - tem seu início durante a Segunda Guerra Mundial, em 1944, quando surgiu a 1ª geração de computadores. Inicialmente os computadores eram gigantes, estáticos, possuíam funcionamento eletromecânico, repletoS de metais e com poucas pessoas capazes para operacionalizá-lo e interpretá-lo. Sem a existência da internet, essas máquinas eram utilizadas unicamente para cálculos estratégicos e depois foram sendo melhores desenvolvidas, adquirindo também, posteriormente, a função de comunicação.

Na 1ª geração (1944 - 1958) podemos destacar os computadores Harvard Mark I (1944), Colossus e ENIAC (1946). Na 2ª geração (1959 - 1965), destacamos a diminuição dos tamanhos das máquinas e a existência de transitores. Na 3ª geração (1965 - 1970) há tela e teclado e, a grande inovação, conexão em rede. A 4ª geração (iniciada em 1971) traz computadores portáteis, mouse, luzes nos cabinetes e smartphones. Há ainda debates em torno do tema alegando que já estamos vivendo a 5ª geração de computadores que é composta, além de outras características, por AI, metaverso e moedas digitais.

Dentro do contexto dos computadores, surge então a internet que é o meio pelo qual um computador é ligado à uma rede mundial contendo outros computadores, enviando e obtendo informações e estabelecendo comunicação. Seu surgimento é repleto de desafios e barreiras, mas seu desenvolvimento é potencializado diante desses mesmsos desafios e barreiras. Tal desenvolvimento fez com que ageências e departamentos fossem criados unicamente voltados à tecnologia de informações (DARPA e Arpanet) e, aqui, surgem nomes importantes para o avanço desta tecnologia, tais como Joseph Carl Robnett Licklider, Vinton Cerf, Robert Kahn e Tim Berners-Lee. Outro nomes também se destacam nos avanços tecnológicos como os de Marc Andreessen, Bill Gates e Steve Jobs. Há ainda um destaque memorável a Grace Murray Hopper, uma das primeiras programadoras do Mark I.

O que são Clients?

Início

A grosso modo, Clients são os usuários de computadores que solicitam informações - via internet -, cujo tais informaçãos são requisitada em um Server que retornará a busca com o desejado pelo usuário. Essas informaçãos são retornadas por documentos denominados de Cache 1 que podem conter imagens, HTMLS, CSS, Javascript, entre outros. A leitura desses documentos, pelo Client, só é possível devido aos browser's , ou "navegadores", que comulmente já estão pré-instalados em um computador. Além disso, os navegadores seguem um padrão, tendo diferenças em detalhes que não implicam na leitura dos arquivos. Há ainda as aplicações web que auxiliam nessa comunicação entre Client e Server.

"A aplicação web diz respeito a uma solução que é executada diretamente no browser (ou navegador), não sendo preciso realizar uma instalação na máquina do usuário. Pode-se, também, utilizar como definição 'tudo aquilo que é processado em um servidor terceiro'." (Cairo Noleto)

Na atualidade, podemos também relacionar como ferramenta de acesso à informação - pelo Client - os dispositivos móveis (notebooks, tablets, smartphones e smartwatchs). Os dispositivos móveis já são bastante utilizados e costumam possuir a vantagem da praticidade. A possibilidade de acessar uma determinada página na web de qualquer lugar é um dos princípais benefícios.

O que são Servers?

Início

O "Client" não existe sozinho. Ele é o requisitante. Mas, para que a requisição do Client seja completada com sucesso, o que ele deseja obter precisa buscar a informação em um repositório. Esses repositórios são os ambientes que possuem a informação que será disponibilizada, ou seja, o Server. A conexão com o Server pode ser externa, através da internet, mas também pode ser interna, dentro de uma organização, por exemplo, e essa conexão é realizada através de rede local denominada LAN - Local Area Network. Diferentemente do Client que é utilizado por um único usuário, os Servers podem ser acessado de vários Clients simultaneamente.

Os servidores podem ser de arquivos, segurança (Firewall), streaming, e-mail ou web. Cada servidor é composto por Softwares - que são sistemas operacionais, monitoramento e servidores web -, e por Hardwares - que englobam os armazenamentos (discos rígidos, SSDs), memória e processadores. Sabendo-se disso, é importante também entender que o servidor possui um sistema operacional e, nesse sistema operacional é instalado um Servidor Web que vai atender a requisição dos dados desejados pelo Client. Dentre os sistemas operacionais mais conhecidos estão o Windows Server, Ubuntu, CentOS, Debian, Fedora e o Oracle Linux. Já entre os servidores web - que são instalados nos sistemas operacionais - há o APACHE e o NGINX.

Ampliando um pouco mais o assunto sobre os tipos de servidores, existe o proxy - servidor responsável por restringir o acesso a sites maliciosos ou proibidos em uma determinada rede. Este servidor também pode servir como servidor de autenticação, onde o usuário precisará digitar seu login e senha e o servidor buscará as informações da web. Outro tipo de servidor é o Firewall, que é um servidor de segurança, mas voltado à acessos externos, onde o mesmo realiza o bloqueio de tentativas de conexões maliciosas. O Web Server e o E-mail Server são servidores que possuem funções de armazenamentos de dados sendo que um armazena sites e o outro e-mails, respectivamente. O Database Server é o servidor que possui o banco de dados onde um outro servidor, em que a aplicação está instalada, faz a requisição dos dados solicitados, podendo ser através da internet - com conexão externa - ou sem acesso a internet - por um site interno, em rede corporativa - denominado de Intranet. Por fim, há ainda o servidor DNS - Domain Name Service, que é um tipo de servidor que detém uma lista de nomes e IP's (registro único de cada site em código numérico) onde, ao usuário fazer uma requisição na web, o DNS busca em sua relação a qual TCP-IP 2 está relacionado o site requisitado e o redireciona a tal.

Diante da abordagem principal desta formação, precisamos também aprofundar o conhecimento inerente à DNS ao que diz respeito a hospedagem e domínio de sites. A hospedagem é a disponibilização dos arquivos do seu site em um servidor (próprio ou contratado) que fica na internet para que os Clients possam acessar. Diante dos benefícios, a contratação torna-se viável, pois, a depender do plano de hospedagem, o servidor hospeda inúmeros sites. Outros detalhes possíveis de avaliação nos planos são algumas funcionalidades como a multiplataformas 3, possibilidade do registro de domínio (DNS), conexões seguras (HTTPS), se também é um servidor web, e-mail e/ou streaming. A avaliação do plano, do seu custo-benefício, dependerá do que o contratante deseja. Já o domínio está relacionado à sua propriedade sobre o nome (marca), e-mail e endereço web na internet. Para isso, é necessário o registro do mesmo desde que haja disponibildade desse nome, ou seja, desde que nenhuma outra pessoa, empresa ou organização já o tenha utilizado. Há um custo para o registro desse domínio e para a sua manuntenção anual, e o mesmo pode ser feito antes mesmo de existir um servidor para a hospedagem, bem como o DNS pode ser configurado posteriormente através do plano de hospedagem (se.com) ou pelo Registro.br (se registro brasileiro). Sendo assim, o registro do nome é o primeiro procedimento a ser adotado por quem deseja possuir um site na internet e e-mails corporativos.

Após registrar seu nome e configurar o DNS, esse novo site agora precisará ficar disponível para ser acessado na web, e isso é feito através do FTP - File Transfer Protocol. O FTP é uma conexão para a transferências de arquivos - envio e recebimento -, para o servidor que hospeda o site. Ao contratar o plano de hospedagem, o contratante recebe um endereço para a conexão FTP, um usuário e senha. Também é necessário um software para a realização desta conexão FTP, como por exemplo o Filezilla 4. Ao utilizar o software com o usuário e senha, é possível enviar os arquivos do novo site. Após finalizar o envio dos arquivos, o novo web site poderá ser transferido para o servidor de hospedagem sendo assim possível acessá-lo.

Sobre Linguagens de Programação Web

Início

As aplicações web e/ou aplicativos, para serem utilizados pelos Clients, precisam seguir, sobretudo, comandos e regras. Esses comandos e regras precisam ser descritos e a essa descrição chamamos de Linguagem. A linguaguem escrita em softwares que resultaram na funcionalidade das aplicações web e/ou aplicativos.

"(...) se refere a um conjunto de regras e instruções que um programador constrói para gerar programas e softwares que serão processados por um computador, dispositivo móvel ou outro equipamento." (Tatyane Mendes)

Não há somente uma linguaguem de programação, mas existem as que mais estão sendo utiliadas para desenvolvimentos de projetos e, segundo o RedMonk 5, as 5 linguagens de programação mais utilizadas entre setembro de 2012 e janeiro de 2022 são:

  1. Javascript
  2. Python
  3. Java
  4. PHP
  5. C#

Apesar do termo linguagem de programação ser genérico, há diferença entre o tipo de linguagem que é utilizada no Server e o que é utilizado no Client. Atualmente, sendo bastante utilizadas nos servidores existem as linguagens Javascript, Java, PHP, C# e Ruby, cada uma com seu modo particular de ser escrita. Para os clientes que utilizam aplicações web - por navegadores -, a que tem sido mais popularmente utilizada é o Javascript, mas há também o HTML e o CSS.

Visto que o Javascript opera tanto do lado do servidor quanto do cliente, é importante salientar que é a escrita do código que se dá de forma diferente. Para a execução do que foi escrito é utilizado o mesmo mecanismo dos dois lados, sendo esse mecanismo o V8 6. Para o cliente, onde é utilizado o navegador para consumir os dados, o módulo V8 está integrado ao software do navegador para ler o que foi escrito, no servidor, em Javascript. Por outro lado, no servidor, o V8 utiliza o suporte do NodeJS 7. Em resumo, quem executa de fato o Javascript é o mecanismo V8. E aqui então fica a compreensão do porquê o Javascript tem se destacado como uma das linguagens de programação mais populares.

Apesar de o HTML ter sido relacionado, anteriormente, como uma das linguagens de programação utilizadas no lado do Client, há defensores que o define apenas como um arquivo de marcação, que "é aquela projetada para definir e apresentar textos", onde "os elementos são marcados utilizando tags que explica o objetivo dessa parte do conteúdo". Basicamente é uma estrutura de texto que o navegador consegue interpretar.

Introdução ao HTML na Prática

Para escrever códigos a serem utilizados por navegadores, a quem deseja iniciar no mundo da programação, é indicado a utilização do software Visual Studio Code 8 e, para os testes de desenvolvimento do projeto, o navegador Google Chrome 9, ambos disponíveis gratuitamente. É possível ainda, para o desenvolvimento de projetos em HTML, utilizar editores de textos simples como o Bloco de Notas, Sublime, NotePad++, entre outros. No entanto, no Visual Studio Code existem diversas funcionalidades que potencializam o resultado do projeto, tais como a instalação de plugins e possibilidade de visualização do projeto à medida que este é desenvolvido. Quanto ao navegador, o Google Chrome também possui uma gama de recursos onde é possível analisar o resultado do projeto já desenvolvido, como, por exemplo, o inspetor de elementos, que apresenta como a página consultada foi construída e quais os elementos foram utilizados para a formatação dessa construção.

Seguindo a lógica de uma estrutura de redação, onde o texto precisa ter a introdução o desenvolvimento e a conclusão, uma página web (HTML) também possui uma estrutura de configuração a ser seguida afim de que o projeto funcione corretamente. Os elementos principais, essenciais e indispensáveis de uma estrutura básica de página web são:

  • html: é toda a composição de dados e informações presente no site, sendo elas visíveis ou não ao usuário;
  • head: informações carregadas para o usuário antes da página ser completamente processada, normalmente visíveis na aba do navegador;
  • body: é o corpo de informações do site que são visíveis ao usuário.

Essa estrutura formam as páginas dos sites, com suas diversas abas, sendo a página principal, dentro do projeto, conhecida como index.

Certamente, uma página web não se limita apenas à estrutura anteriomente apresentada, mas possui inúmeros outros elementos que auxiliam na funcionalidade, estrutura, organização e disposição dessa página. A esses elementos chamamos de tags. As tags são marcações e instruções de como o conteúdo do site (texto, imagem, vídeos, links) deverão aparecer no navegador quando acessado pelo usuário. Elas são responsáveis por delimitar o que cada código deve executar na página web. Além disso, algumas tags possuem atributos - que são as características das tags, tais como cor, tamanho e espaço. É importante ainda citar que, dentro de um projeto de site que possuirá textos, algumas tags podem ser essenciais, como as que são destinadas à títulos e subtítulos, parágrafos e citações, bem como as que são destinadas à listas - ordenadas ou não - e que, consequentemente, necessitam da tag destinada à itens para funcionar corretamente.

Por fim, uma página web é criada para ser - muito - navegada. Nesse sentido, elas precisam gerar a interação com o cliente e tal interação pode ser muito bem fomentada através dos links disponíveis no site. Para tanto, também existem tags específicas para facilitar a navegação dentro do próprio site ou em sites externos.
